Webhooks allow you to receive real-time notifications about events in your Ultravox account. You can configure webhooks to send HTTP POST requests to a specified URL when certain events occur.

## Webhooks in Ultravox
When creating a webhook, you must provide a URL that will receive the webhook notification. Your service must return an HTTP status code of 200 ("OK"). If your service fails to respond with a 200, we will retry sending again.

### Retries
When a destination URL fails to respond to an incoming webhook with a 200, we will retry using an exponential backoff strategy as follows:

* Maximum of 10 retries.
* First retry will occur 30 seconds later.
* Subsequent retries will double the retry interval. Last retry will be 8 hours later.

If your receiving service is still down, you can use the API to retrieve information about any calls that occurred.

## Available Events
The following events are available and can be specified when creating or updating a webhoook.

| event        | description                |
|------------- | -------------------------- |
| call.started | Fired when a call starts.  |
| call.ended   | Fired when a call ends.    |
